# sifuuuuto

管理者ログインとサーバー保存に対応したシフト管理サイトです。

## 構成

- `index.html`: シフト表の UI
- `styles.css`: 表示スタイル
- `script.js`: 管理者ログイン、保存、サーバー同期ロジック
- `server.js`: スケジュール保存用の Node.js サーバー
- `schedule.json`: 保存済みのスケジュールデータ
- `package.json`: 依存関係と起動スクリプト

## 使い方

1. サーバーを起動します:
   ```bash
   cd /workspaces/sifuuuuto
   npm install
   npm start
   ```
2. ブラウザで `http://localhost:3000` を開きます。
3. デフォルトは閲覧モードです。右上の「管理者ログイン」ボタンを押すと管理者モードに入ります。
4. 閲覧モードでは、管理者にシフト変更のリクエストを送信できます。
5. 管理者ログイン後は、日付・役職・番号を選択してセルをクリックし、担当を割り当てます。
6. 「変更を保存」をクリックすると、`schedule.json` に保存されます。

## 他の端末から見る

サーバーを同じネットワーク上で公開している場合、他の端末からも `http://<サーバーのIP>:3000` を開くと最新の結果を表示できます。サーバーが起動している限り、15秒ごとに最新データを自動取得します。

## 管理者モード

- 管理者パスワード: `passward`
- 管理者ログイン後に編集と保存が可能です

## 注意

- 現在はシンプルなパスワード認証です。公開環境にデプロイする場合は、HTTPS や安全な認証を追加してください。

## デプロイ

### Docker での公開

1. ビルド
   ```bash
   docker build -t sifuuuuto .
   ```
2. 起動
   ```bash
   docker run -p 3000:3000 sifuuuuto
   ```
3. ブラウザで `http://localhost:3000` にアクセスします。

### Heroku での公開

1. `Procfile` を使ってデプロイできます。
2. Heroku アプリを作成し、Git リポジトリをプッシュしてください。

### Render での公開

このリポジトリには `render.yaml` を追加済みです。Render に GitHub 連携している場合、`main` ブランチをデプロイすると自動的にビルドされます。
